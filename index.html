Introduction

In traditional Finite State Machines you have only one level of nesting: a State Machine contains States.

In Hierarchical State Machines a State Machine can also be one of the sub-states of a containing State Machine. The problem of implementing a complex State Machine can thus be simplified by:

  • implementing a set of simpler sub State Machines;
  • wiring together these sub State Machines in a containing State Machine (or an arbitrarily deep hierarchy of sub State Machines).
The current library implements such a Hierarchical State Machine. It is also a project I use to learn Scala (my very first lines of Scala) and therefore the source code probably looks more imperative that it might be. Comments are most welcome on how to make it more functional.
